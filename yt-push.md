저장된 유튜브 요약본을 **"무슨 내용이냐"에 맞춰** 개인 git repo(공부 노트)에 발행한다.
"유튜브에서 왔다"가 분류 기준이 아니다 — **내용 주제로 분류**하고, 관련 주제 폴더가 있으면 그걸 확장(재사용),
없으면 적절한 이름으로 새로 만든다. (네가 직접 판단해서 적재한다.)

## 절차
1. **대상 요약본 결정**
   - `$1`(경로)이 주어지면 그 파일.
   - 비어 있으면 `yt-script --latest` 를 실행해 `LATEST_FILE:` 경로를 쓴다.
2. **내용 파악**: 그 `.md` 를 Read 한다(직전에 `/yt` 로 막 만들어 내용을 이미 안다면 생략 가능).
   제목·도메인·본문으로 이 영상이 *무슨 주제*인지 한 줄로 정리한다.
3. **기존 주제 폴더 확인**: `yt-script --topics` 를 실행해 발행 루트의 기존 폴더 목록(`TOPICS:`)을 본다.
4. **주제 폴더 결정** (핵심 판단)
   - 기존 폴더 중 내용과 의미가 맞는 게 있으면 **그 이름을 철자·대소문자·하이픈까지 그대로** 재사용한다(확장).
     예: 목록에 `claude-code` 가 있고 영상이 Claude Code 관련이면 → `claude-code`.
   - 맞는 폴더가 없으면 **기존 폴더의 작명 컨벤션**(예: 영문 소문자-하이픈, 또는 한국어 명사형)을 따라
     간결한 새 이름을 정한다. 너무 일반적(`misc`)이거나 영상 한 편짜리 과도하게 좁은 이름은 피한다.
   - 정말 애매하면 `미분류`.
5. **발행**: `yt-script --publish "<요약본 경로>" "<결정한 주제 폴더>"` 를 실행한다.
   - 없는 폴더는 yt-script 가 자동 생성하고, 폴더명은 안전하게 정규화된다.
6. **결과 보고**: 출력의 `PUSHED_FILE:`(성공) 또는 `COMMITTED_FILE:`(커밋됐으나 push 실패) 경로와 함께,
   **어느 폴더에 / 기존 재사용인지 신설인지 / 왜 그렇게 분류했는지**를 한 줄로 설명한다.

## 예외 처리
- `YT_PUBLISH_DIR 가 설정돼 있지 않습니다`: 발행 루트를 환경변수로 지정해야 한다고 안내한다.
  예: `export YT_PUBLISH_DIR="$HOME/Desktop/dev-lab/study"` (해당 repo는 미리 git clone 되어 있어야 함).
- `변경 없음`: 이미 동일 내용이 올라가 있다는 뜻 — 그대로 사용자에게 알린다.
- `push 실패`(COMMITTED_FILE): 네트워크/인증 문제일 수 있으니 잠시 후 재시도하거나 해당 repo에서 직접 `git push` 하도록 안내한다.
