저장된 유튜브 요약본을 **"무슨 내용이냐"에 맞춰** 개인 git repo(공부 노트)에 발행한다.
폴더는 내용 주제로 정하고(관련 폴더 있으면 확장, 없으면 신설), **파일명도 날짜·영상ID 없이 깔끔한 제목 슬러그**로 짓는다.

## 절차
1. **대상 요약본 결정**
   - `$1`(경로)이 주어지면 그 파일. 비어 있으면 `yt-script --latest` 의 `LATEST_FILE:` 경로.
2. **내용 파악**: 그 `.md` 를 Read 한다(직전에 `/yt` 로 막 만들어 내용을 이미 안다면 생략 가능).
   제목·도메인·본문으로 *무슨 주제*인지 정리한다.
3. **기존 주제 폴더 확인**: `yt-script --topics` 로 발행 루트의 기존 폴더(`TOPICS:`)를 본다.
4. **주제 폴더 결정**
   - 기존 폴더 중 의미가 맞는 게 있으면 **그 이름을 철자·대소문자·하이픈까지 그대로** 재사용한다(확장).
     예: 목록에 `claude-code` 가 있고 영상이 Claude Code 관련이면 → `claude-code`.
   - 없으면 기존 폴더 작명 컨벤션(영문 소문자-하이픈 또는 한국어 명사형)을 따라 간결한 새 이름을 정한다.
     너무 일반적(`misc`)이거나 한 편짜리로 과도하게 좁은 이름은 피한다. 정말 애매하면 `미분류`.
5. **파일명(제목 슬러그) 결정**
   - `yt-script --ls "<결정한 주제 폴더>"` 로 그 폴더의 기존 파일명(`FILES:`)을 확인한다.
   - 내용을 대표하는 **짧고 읽기 좋은 슬러그**를 짓는다 — **날짜·영상ID 빼고**, 기존 파일들의 스타일을 따른다.
     예: `claude-code-실전-치트시트.md` 처럼 `소문자-하이픈`(또는 한국어는 `명사-하이픈`) 형태. 확장자 `.md` 포함.
   - 기존 파일과 **이름이 겹치지 않게** 한다(겹치면 같은 파일을 덮어써 갱신하는 게 의도일 때만 그대로, 아니면 구분되는 이름으로).
6. **발행**: `yt-script --publish "<요약본 경로>" "<주제 폴더>" "<파일명.md>"` 를 실행한다.
   - 없는 폴더는 자동 생성되고, 폴더명·파일명은 안전하게 정규화된다(공백→하이픈, 허용문자만).
7. **결과 보고**: `PUSHED_FILE:`(성공) / `COMMITTED_FILE:`(커밋됐으나 push 실패) 경로와 함께,
   **어느 폴더에 / 어떤 파일명으로 / 기존 재사용인지 신설인지 / 왜 그렇게 정했는지**를 한 줄로 설명한다.

## 예외 처리
- `YT_PUBLISH_DIR 가 설정돼 있지 않습니다`: 발행 루트를 환경변수로 지정해야 한다고 안내한다.
  예: `export YT_PUBLISH_DIR="$HOME/Desktop/dev-lab/study"` (해당 repo는 미리 git clone 되어 있어야 함).
- `변경 없음`: 이미 동일 내용이 올라가 있다는 뜻 — 그대로 사용자에게 알린다.
- `push 실패`(COMMITTED_FILE): 네트워크/인증 문제일 수 있으니 잠시 후 재시도하거나 해당 repo에서 직접 `git push` 하도록 안내한다.
